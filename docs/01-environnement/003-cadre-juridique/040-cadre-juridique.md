# Un cadre juridique

## Types d’infractions

*   **Définition :** Les différentes catégories d'actions illégales en matière de sécurité informatique.
*   **Exemples :**
  *   Accès non autorisé à un système informatique (Art. 323-1 du Code pénal).
  *   Vol de données  (Art. 323-3 du Code pénal).
  *   Modification ou destruction de données(Art. 323-3 du Code pénal).
  *   Interception de communications (Art. 226-15 du Code pénal).
  *   Diffusion de logiciels malveillants (Art. 323-3-1 du Code pénal).
*   **Conséquences légales :**
  *   Sanctions pénales (amendes, emprisonnement).
  *   Sanctions civiles (dommages et intérêts).

## Un problème de classification

*   **La difficulté de catégoriser les infractions informatiques :**
  *   Évolution rapide des technologies et des menaces.
  *   Complexité des systèmes informatiques.
  *   Besoin d'une expertise technique pour qualifier les faits.
*   **L'importance d'une classification précise pour :**
  *   Adapter les lois et réglementations.
  *   Faciliter les enquêtes et les poursuites.
  *   Sensibiliser les utilisateurs et les professionnels.
